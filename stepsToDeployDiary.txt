🧩 מקרה 1 — העלאה ראשונה של Submodule לפרודקשן

כאשר זו הפעם הראשונה שאתה מכניס את mongoose (או כל submodule אחר) לפרודקשן.

🪜 שלבים
1️⃣ 
cd /path/to/Diary


📍 היכנס לתיקיית הפרויקט בפרודקשן (שם יש את server.js ו־.git).

2️⃣ משיכת קוד עדכני מהריפו הראשי
git pull origin master


מושך את הגרסה האחרונה של Diary מה־GitHub.
אם app/shared/mongoose עוד לא קיים, הוא עדיין לא יורד בפועל (רק הרפרנס).
3️⃣ אתחול ה־submodules לראשונה
git submodule init
git submodule update --recursive


🚀 יוצר את תיקיות ה־submodules ומוריד את הקוד שלהן (mongoose, login, pdf וכו׳).
זה חובה בהעלאה ראשונה בלבד.

4️⃣ latest commit (רשות, אך מומלץ)
git submodule update --remote --merge


מושך את ה־commit הכי חדש מה־GitHub של כל submodule,
כך שגם אם Diary מצביע על גרסה ישנה — תעבוד עם האחרונה.

5️⃣ 
npm install

	- rm -rf ./dist 			// this is to remove dist folder (only in case it exsist)
	- npm run build:production	
	- pm2 kill 			// kill all existing processes
	- pm2 serve ./dist --name Diary --port 8084		// to serve the frontend new build

מתקין את כל ה־packages (כולל mongoose, express וכו׳).

אם יש client נפרד:

cd client
npm install
cd ..

6️⃣ הפעלת השרת
	pm2 start server.js  --name DiarySERVER

או אם כבר קיים:
pm2 restart diary


🔄 עכשיו Diary רץ עם ה־mongoose בפעם הראשונה.

🔁 מקרה 2 — עדכון עתידי של mongoose (כבר קיים בפרודקשן)

כאשר כבר יש mongoose בפרודקשן,
ורק דחפת commit חדש ל־GitHub ורוצה לעדכן את הקוד.

🪜 שלבים
1️⃣ 
cd /path/to/Diary

2️⃣ משיכת הקוד הראשי
git pull origin master


📦 מביא את ה־Diary המעודכן, כולל ההפניה החדשה ל־mongoose.

3️⃣ עדכון כל ה־submodules לגרסאות האחרונות
git submodule update --remote --merge


🔄 מושך את ה־commits האחרונים מכל submodule שמוגדר ב־.gitmodules.

4️⃣ התקנת תלותים (אם נוספו)
npm install


אם ביצעת שינוי בקוד שהוסיף או הסיר חבילות npm.

5️⃣ הפעלת השרת מחדש
pm2 restart diary


♻️ מרענן את היישום עם גרסת mongoose החדשה.

6️⃣ 

בדוק דרך הדפדפן או Postman שה־API של mongoose עובד:

http://your-server-ip:3004/api/generic/entity?model=holder

🧠 טיפ חשוב — לזהות אם צריך init או update

תריץ:

git submodule status


אם מופיע סימן - בתחילת השורה:

-a1b2c3d4 app/shared/mongoose


זה אומר שה־submodule עדיין לא מאותחל
→ צריך להריץ git submodule init.

אם מופיע סימן +:

+a1b2c3d4 app/shared/mongoose


זה אומר שיש commit חדש במודול
→ צריך להריץ git submodule update --remote --merge.
